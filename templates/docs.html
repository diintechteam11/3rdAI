Core Modules

  • Camera Registry

    Inventory and identification of all connected RTSP/IP units.

  • Recording Control

    Manual and automated start/stop management.

  • Scheduler

    24/7 or custom time-window recording profiles.

  • Analysis Trigger

    Auto-start capture synchronized with AI detection firing.

DB Requirements

Every transaction is stored with UTC ISO 8601 timestamps for audit compliance. The system ensures 100% data traceability from trigger to video file.

System Overview

The Camera Recording API acts as the central control plane for multi-camera video infrastructures. It enables developers to build interfaces for live surveillance management, secure video storage, and AI-driven reporting.

Manual Mode

On-demand capture for incident recording and manual monitoring.

Analysis Mode

Synchronized auto-capture triggered by Computer Vision events.

Storage Architecture

Local / Hot Storage

Buffered recordings and processed frames are stored in the /static/recordings directory for instant playback.

Cloud Archive (R2/S3)

AI detection crops are automatically offloaded to Cloudflare R2 for long-term accessibility and global CDN delivery.

The 3rdAI platform operates under a zero-trust architecture model. All recording paths are cryptographically signed and tracked via unique task identifiers.
